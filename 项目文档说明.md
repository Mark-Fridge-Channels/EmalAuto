# M365 Mail Orchestration System

## 项目需求文档（PRD） + 技术开发文档（Technical Design）

---

# 一、项目目标

开发一个：

```text id="w7p3mx"
基于 Microsoft Graph API 的
多域名 / 多邮箱 邮件编排系统
```

系统需要：

* 管理多个 M365 域名
* 管理多个邮箱账号
* 通过程序自动发信
* 自动监听回信
* 自动识别退信（Bounce）
* 自动进行线程匹配（Conversation Matching）
* 与 Notion Database 双向同步

---

# 二、核心业务流程

---

# 1. 发信流程

```text id="y4m8pc"
Notion Database
    ↓
Task Polling
    ↓
Mail Send Queue
    ↓
Microsoft Graph API
    ↓
Exchange Online
    ↓
保存 message metadata
    ↓
回写 Notion
```

---

# 2. 收信流程

```text id="w2n5tr"
Mailbox Inbox
    ↓
Polling / Webhook
    ↓
Delta Query
    ↓
获取最新邮件
    ↓
Reply Matching
    ↓
识别回复 / Bounce
    ↓
回写 Notion
```

# 十一、最终建议（重要）

你现在最正确路线：

```text id="m7n4wc"
Graph-only
+
Polling-first
+
ConversationId tracking
```

不要一开始：

* webhook
* 多 provider
* IMAP
* SMTP abstraction

否则复杂度会暴涨。
